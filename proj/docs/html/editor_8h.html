◆ editor_remote_delete_char()

EditorResult editor_remote_delete_char ( )

Deletes the character immediately before the remote cursor position.

If the remote cursor is at the beginning of a line, it merges the current line with the previous line, performing a memmove to collapse the rows array. Works similarly to editor_delete_char but without updating the local cursor position.

Returns
EDITOR_OK on success, or EDITOR_ERR_ALLOC_FAILED on memory error.

◆ editor_remote_insert_char()

EditorResult editor_remote_insert_char ( char c)

Inserts a character at the remote cursor position.

Manages the memory reallocation if the line needs to grow, and pushes the text right. If a newline character is inserted, it splits the current line dynamically. The implementation assumes that the cursor is always within the bounds of the document and it is very similar to editor_insert_char, without updating the local cursor position.

Parameters
cThe character to insert.
Returns
EDITOR_OK on success, or EDITOR_ERR_ALLOC_FAILED on memory error.

◆ editor_remote_replace_block()

EditorResult editor_remote_replace_block ( int start_row,
int deleted_count,
int inserted_count )

Prepares the text buffer for a block replacement.

This is the core of the remote block processing algorithm. It handles the structural memory shifts (memmove on the lines array) required when multiple lines are deleted or inserted at once. For example, during a Paste or Cut operation, it dynamically expands or shrinks the total row count of the buffer.

Parameters
start_rowThe index of the first row to be affected.
deleted_countNumber of lines to remove from the buffer.
inserted_countNumber of empty lines to allocate and make room for.
Returns
EDITOR_OK on success, or EDITOR_ERR_ALLOC_FAILED on memory error.

Overwrites a specific line with new text received from the network.
